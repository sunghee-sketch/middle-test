# 하이퍼파라미터 튜닝 전략

---

## 1. 현재 문제가 뭔가요?

모델을 학습시키고 성능을 측정했더니 이런 결과가 나왔습니다:

- 연습문제 점수 (CV R²): **0.57**
- 실전 시험 점수 (Test R²): **0.36**

쉽게 말하면, **연습할 때는 57점인데 실전에서는 36점**이 나온 것입니다.
이런 현상을 전문용어로 **과적합(Overfitting)** 이라고 합니다.

> 과적합 = 모델이 연습 데이터만 너무 외워버려서, 새로운 데이터에는 제대로 적용하지 못하는 현상

---

## 2. 왜 과적합이 생겼나요?

모델에는 여러 **설정값(하이퍼파라미터)** 이 있는데, 현재 설정이 너무 "열심히 외우도록" 되어 있기 때문입니다.

예를 들어:
- `max_depth=5` → 모델이 너무 복잡한 패턴까지 외움 (깊이를 줄여야 함)
- `reg_alpha=1.0`, `reg_lambda=5.0` → 외우는 것을 막는 제약이 너무 약함 (더 강하게 해야 함)

> 하이퍼파라미터 = 모델의 "설정값". 요리로 비유하면 불 세기, 조리 시간 같은 것.
> 정규화(Regularization) = 모델이 너무 많이 외우지 못하도록 제약을 주는 것.

---

## 3. 어떻게 좋은 설정값을 찾나요?

### 방법 비교

| 방법 | 쉬운 설명 | 전문용어 |
|------|-----------|----------|
| 모든 조합 다 시도 | 메뉴판에서 모든 조합 주문해보기 | Grid Search |
| 랜덤으로 골라서 시도 | 눈 감고 메뉴판 찍기 | Random Search |
| 결과 보면서 점점 좁혀가기 | 맛있는 쪽으로 계속 조정하기 | Bayesian Optimization |

**이번에 선택한 방법: Bayesian Optimization (Optuna 라이브러리 사용)**

- 처음엔 여러 설정값을 시도해보고
- "이 방향이 좋았다"는 것을 기억해서
- 점점 더 좋은 설정값 쪽으로 탐색을 좁혀갑니다
- 50번만 시도해도 전체 탐색(6,561번)에 가까운 결과를 얻을 수 있습니다

---

## 4. 어떤 설정값을 탐색하나요?

| 설정값 | 쉬운 설명 | 탐색 범위 | 방향 |
|--------|-----------|-----------|------|
| `max_depth` | 모델이 패턴을 얼마나 복잡하게 학습하는지 | 2~5 | 낮출수록 과적합 감소 |
| `min_child_weight` | 학습에 필요한 최소 데이터 수 | 5~30 | 높일수록 과적합 감소 |
| `reg_alpha` | 불필요한 변수를 0으로 만드는 제약 강도 | 1~20 | 높일수록 과적합 감소 |
| `reg_lambda` | 전체 변수의 영향력을 줄이는 제약 강도 | 5~50 | 높일수록 과적합 감소 |
| `subsample` | 학습할 때 데이터를 몇 %만 사용하는지 | 0.4~0.7 | 낮출수록 다양성 증가 |
| `colsample_bytree` | 학습할 때 변수를 몇 %만 사용하는지 | 0.3~0.5 | 낮출수록 다양성 증가 |
| `learning_rate` | 한 번 학습할 때 얼마나 크게 수정하는지 | 0.005~0.02 | 낮을수록 신중하게 학습 |

---

## 5. 학습을 언제 멈추나요?

`n_estimators=2000`으로 설정했지만, 실제로는 2000번 다 학습하지 않습니다.
검증 점수가 50번 연속으로 나아지지 않으면 자동으로 멈춥니다.

> 이것을 **Early Stopping(조기 종료)** 이라고 합니다.
> 쉽게 말하면, "더 이상 나아지지 않으면 그만하자"는 것.

---

## 6. 최종 설정값은 어떻게 고르나요?

단순히 점수가 제일 높은 것을 고르지 않고, **안정성**도 함께 고려합니다.

```
선택 기준 = 평균 점수 - (0.5 × 점수 편차)
```

예를 들어:
- A 설정: 평균 0.65, 편차 0.10 → 점수 = 0.65 - 0.05 = **0.60**
- B 설정: 평균 0.63, 편차 0.02 → 점수 = 0.63 - 0.01 = **0.62** ← 선택

> 이것을 **안정성 우선 선택 기준** 이라고 합니다.
> 들쭉날쭉한 성능보다 꾸준히 좋은 성능이 실전에서 더 믿을 수 있기 때문입니다.

## 8. 실제 튜닝 결과

### Optuna가 찾은 최적 설정값

| 설정값 | 이전값 | 최적값 | 변화 |
|--------|--------|--------|------|
| `max_depth` | 5 | **3** | 트리 깊이 축소 |
| `min_child_weight` | 5 | 5 | 동일 |
| `reg_alpha` | 1.0 | **2.07** | 정규화 강화 |
| `reg_lambda` | 5.0 | **15.91** | 정규화 대폭 강화 |
| `subsample` | 0.7 | 0.65 | 소폭 감소 |
| `colsample_bytree` | 0.5 | 0.41 | 감소 |
| `learning_rate` | 0.02 | 0.015 | 소폭 감소 |

### 성능 비교

| 항목 | 이전 | 튜닝 후 | 변화 |
|------|------|---------|------|
| CV R² (연습 점수) | 0.5655 | **0.5346** | -0.031 |
| Test R² (실전 점수) | 0.3570 | **0.4831** | **+0.126** |
| 두 점수 차이 | 0.208 | **0.052** | **대폭 감소** |

> 연습 점수는 약간 낮아졌지만, 실전 점수가 크게 오르고 두 점수 간 격차가 줄어든 것이 핵심 성과입니다.
> 모델이 덜 외우는 대신 새로운 데이터에도 잘 적용되도록 개선됐습니다.

### 예상 대비 실제 결과

| 항목 | 예상 | 실제 |
|------|------|------|
| Test R² | 0.50~0.58 | **0.4831** |
| 두 점수 차이 | 0.10~0.15 | **0.052** |

- Test R²는 예상 범위 하단에 근접했고, 두 점수 격차는 예상보다 더 잘 줄어들었습니다.
