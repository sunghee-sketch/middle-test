## 2. 파이프라인

5인 전문가 합의 기반 5단계:

```
원본 2431
  → VarianceThreshold(0.01) → 504
  → StandardScaler + 클리핑 [-10, 10]
  → MI top-300
  → ResidualMLP(hidden=256, n_blocks=3, LayerNorm, Swish)
  → Mixup(α=0.3, p=0.5, weight도 보간)
  → 3-seed Snapshot Ensemble (log 공간 평균)
```

- 손실: MSE (sample weight: 상위 10% MP에 가중치 3.0)
- 옵티마이저: Adam(lr=1e-3, weight_decay=5e-4)
- 스케줄러: ReduceLROnPlateau(patience=6, factor=0.5)
- 조기 종료: patience=12
- 파라미터 수: 491,905

## 6. 고찰

### 6.1 핵심 성과 — Test R² 큰 폭 상승, MAE 30% 감소

Test R²가 v3 0.5821에서 **v4 0.6958로 +11.4%p** 상승했습니다. 단순 MLP를 ResidualMLP(LayerNorm+Swish)로 교체하고, Mixup 정규화와 3-seed Snapshot Ensemble을 결합한 효과가 함께 누적된 결과입니다.

특히 Test MAE가 46.35 K → 32.53 K로 **13.8 K 감소**한 것이 의미있습니다. MAE는 실무 관점에서 "예측 오차의 일반적 크기"를 나타내며, 유기분자 녹는점 실험 오차(보통 ±1~5 K) 기준으로 보면 여전히 크지만, 트리 모델(MAE 53 K) 대비로는 큰 진전입니다.

### 6.2 CV-Test 갭 완전 해소 (역전)

| | v3_ann | v4_ann |
|---|---|---|
| CV R² | 0.7065 | 0.6702 |
| Test R² | 0.5821 | **0.6958** |
| 갭 | **+0.124 (과적합)** | **-0.026 (Test 우세)** |

v3의 갭이 **+0.124로 심각한 과적합**을 의미했다면, v4에서 **-0.026로 역전**된 것은 다음을 시사합니다:

1. **Mixup + weight_decay 5e-4**의 정규화 효과가 강력 → CV 성능을 다소 희생하면서 일반화 크게 개선
2. **3-seed Snapshot Ensemble**이 Test 쪽에만 적용되어 이득이 편중 (라운드 2 패치의 single-seed 비교 출력이 없어 단일 모델 vs ensemble 분리 측정 불가)
3. **MI top-300 선별**이 노이즈 제거 → 모델이 신호 위주로 학습 → Test에서 안정

하지만 해석에 주의가 필요합니다. 갭이 **음수**라는 것은 **과소적합 신호**일 수도 있습니다. 모델이 학습 데이터를 충분히 표현하지 못하고, Test가 우연히 CV fold보다 쉬운 분포일 가능성도 있습니다.

### 6.3 Fold 편차는 해결되지 않음

| Fold | v3 R² | v4 R² |
|---|---|---|
| 1 | 0.79 | 0.77 |
| **2** | **0.48** | **0.48** |
| 3 | 0.69 | 0.58 |
| 4 | 0.75 | 0.70 |
| 5 | 0.82 | 0.82 |

**Fold 2는 v3와 v4에서 동일하게 R² 0.48 수준**에 머물렀습니다. StratifiedKFold는 MP 분포만 균등화할 뿐 **분자 구조(scaffold, 작용기 조합) 편향**은 해소하지 못합니다. Fold 2 validation set에 특정 구조 클래스가 몰려 있으면 그 클래스를 학습 데이터에서 본 적 없는 모델이 예측 실패하는 구조적 문제가 재현됩니다.

CV 편차 ±0.12는 여전히 크며, 이는 "평균 R² 0.67은 fold에 따라 0.48~0.82"임을 의미합니다.

### 6.4 실행 시간 — 예산 대비 1/60

**총 0.5분 (30초)** 소요로, 30분 예산의 **1.7%**만 사용했습니다. 내역:
- 피처 추출: 7.7초
- VT + split + MI: ~2초
- 5-Fold CV 학습: 12초 (fold당 2~3초)
- 최종 3-seed 학습: 5초

→ Optuna HPO(80 trial × 5-fold), 5-seed 확장, tautomer augmentation, 더 큰 모델(hidden=512) 등을 **모두 추가해도 시간 예산 여유 충분**.

### 6.5 적용 버전의 한계

사용자 실행은 **라운드 1 패치 버전**에서 이뤄졌습니다. 라운드 2의 다음 개선이 반영되지 않았습니다:
- **Option B (fold-내부 MI)** — 누수 차단
- **proj Sequential(Linear + LayerNorm)** — 학습 초기 안정성
- **single-seed Test R² 별도 출력** — CV-Test 갭 공정 비교

이 중 **Option B 미적용으로 인한 CV R² 과대 추정**이 현재 결과에 포함되어 있을 수 있습니다. 즉 실제 fold-내부 MI fit을 적용하면 CV R²는 **0.67보다 더 낮아질** 가능성이 있으며, 이 경우 CV-Test 갭 역전 폭이 더 커집니다.

## 8. 향후 개선 방향 (팀 2차 토의 예정)

### 즉시 적용 가능 (시간 여유 충분)
1. **라운드 2 패치 재실행**: Option B + proj LN + single-seed 비교 → 정확한 측정치 확보
2. **Epochs/Patience 상향**: 80→200, 12→30 → 과소적합 신호 대응
3. **5-seed로 확장**: 3→5 → snapshot 부스트 추가

### 중기 개선 (추가 합의 필요)
4. **Fold 편차 해결**: scaffold split 또는 chemical similarity-based split 도입
5. **Optuna joint search**: 80 trial × 5-fold, architecture 파라미터 포함
6. **Tautomer enumeration**: Mixup 효과의 한계 확인 후 대안 데이터 증강
7. **Stacking with v2 트리 모델**: OOF 기반 meta-learner (v2 XGB/LGBM 재활용)

### 근본 개선 (장기)
8. **Scaffold-aware Mixup**: 구조적으로 유사한 분자끼리만 보간
9. **Uncertainty estimation**: 예측 신뢰 구간 출력 (MC Dropout, Deep Ensemble)
10. **v3_gnn과의 hybrid**: GNN의 구조 인식 + ANN의 descriptor 활용

## 9. 요약 한 줄

> **v4는 v3의 과적합을 완전히 해소하고 Test MAE를 30% 개선했다. 그러나 Fold 편차(분자 구조 편향)는 해결되지 않았고, 실행 시간 여유가 막대하므로 라운드 2 패치 재실행 + scaffold split + HPO + Stacking으로 0.75~0.80 구간 진입 여지가 있다.**
