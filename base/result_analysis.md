## 4. 고찰

### 핵심 변화: 과적합 해소

이번 튜닝의 핵심 성과는 **CV R² 향상이 아닌 과적합 해소**입니다.

- CV-Test 격차가 **0.208 → 0.052**로 대폭 감소
- Test R²가 **0.357 → 0.483**으로 0.126 향상

쉽게 말하면, 이전 모델은 "연습문제만 잘 푸는 모델"이었는데, 튜닝 후에는 "실전에서도 잘 푸는 모델"이 됐습니다.

---

### 왜 Test R²가 올랐는가?

**1. max_depth 5 → 3 (가장 큰 원인)**

트리 깊이를 줄인 것이 과적합 해소에 가장 직접적으로 기여했습니다. 깊은 트리는 훈련 데이터의 노이즈까지 외워버리는 경향이 있는데, 깊이를 3으로 줄여 단순하고 일반적인 패턴만 학습하도록 강제했습니다.

**2. reg_lambda 5 → 15.9 (L2 정규화 강화)**

2058개의 고차원 피처에 비해 기존 정규화가 너무 약했습니다. L2 정규화를 3배 이상 강화해 불필요한 피처의 영향력을 억제했습니다.

**3. Melting_point_2.csv 사용**

기존에 파싱 실패로 제외됐던 3개 샘플이 추가되어 훈련 데이터가 소폭 증가했습니다.

---

### 왜 CV R²는 약간 낮아졌는가?

이것은 **정상적인 트레이드오프**입니다.

모델이 덜 외우도록(과적합 억제) 설정했으니, 연습 점수(CV R²)가 약간 낮아지는 것은 당연합니다. 중요한 것은 실전 점수(Test R²)가 올랐다는 점입니다.

---

### 아쉬운 점: CV 편차 증가

CV R² 편차가 ±0.039 → ±0.079로 커졌습니다. 특히 Fold 4(R²=0.415)에서 성능이 크게 떨어졌습니다. 특정 데이터 분할에서 고MP 극단값이 몰렸을 가능성이 있으며, 이는 데이터 불균형 문제가 여전히 남아있음을 시사합니다.

---

### 향후 개선 여지

1. **Optuna 탐색 횟수 증가** (50회 → 200회): 더 넓은 범위 탐색
2. **이상치 제거**: 음수 MP(-348K), 극고온(>1000K) 샘플 제거 시 편차 감소 예상
3. **앙상블**: XGBoost + LightGBM + RandomForest 평균으로 추가 향상 가능
