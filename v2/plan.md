# v2 개선 계획서

## 왜 XGBoost + LightGBM + RandomForest 앙상블을 추천했는가?

세 모델은 각각 **실수하는 패턴이 다릅니다.**

| 모델 | 학습 방식 | 쉬운 비유 |
|------|-----------|-----------|
| XGBoost | 이전 실수를 순서대로 보완하며 학습 | 틀린 문제만 반복 복습 |
| LightGBM | XGBoost와 비슷하지만 더 빠른 방식 | 같은 방식, 더 빠른 속도 |
| Random Forest | 여러 모델이 독립적으로 학습 후 투표 | 여러 사람이 각자 풀고 다수결 |

XGBoost와 LightGBM은 "부스팅", Random Forest는 "배깅"이라는 서로 다른 방식입니다. 한 모델이 틀리는 데이터를 다른 모델이 맞추는 경우가 많아, 셋을 합치면 서로의 실수를 보완합니다.

---

## 현재 모델의 문제점 3가지

에이전트 4명의 토의 결과, 문제를 3가지로 정리했습니다.

---

### 문제 1. 데이터 자체에 문제가 있다 (Agent 3)

**① 물리적으로 불가능한 값이 있다**

MP가 -348K인 샘플이 존재합니다. 절대온도(K)는 0K 아래로 내려갈 수 없으므로 이 값은 잘못된 데이터입니다. 아마도 섭씨(℃)를 켈빈(K)으로 변환할 때 오류가 생긴 것으로 보입니다.

> 잘못된 데이터로 학습하면 모델도 잘못된 패턴을 외우게 됩니다.

**② 고온(1000K 이상) 데이터가 너무 적다**

전체 데이터의 75%가 297K 이하에 몰려 있고, 1000K 이상 샘플은 극소수입니다. 모델 입장에서는 고온 화합물을 거의 본 적이 없으니 예측을 못하는 것이 당연합니다.

> 시험에 나온 유형을 한 번도 연습하지 않은 것과 같습니다.

**③ 데이터 분포가 한쪽으로 치우쳐 있다**

150~400K 구간에 데이터가 몰려 있어 모델이 이 구간에만 최적화되는 경향이 있습니다.

**해결 방법:**
- 음수 MP 샘플 제거 또는 수정
- MP 값에 log 변환 적용 → 분포를 균등하게 펼침 (전문용어: **타깃 변환, Target Transformation**)
- 희소 구간(고MP) 샘플에 더 높은 가중치 부여 (전문용어: **샘플 가중치, Sample Weight**)
- 데이터 분포를 고려한 5-Fold 분할 (전문용어: **Stratified K-Fold**)

**예상 R² 향상: +0.10 ~ +0.20**

---

### 문제 2. 분자를 설명하는 정보가 부족하다 (Agent 2)

**현재 사용 중인 정보:**
- Morgan Fingerprint: 분자 부분구조 2048개 (있다/없다)
- 기술자 10종: 분자량, 극성 등 기본 정보

**문제점:**

녹는점은 고체 상태에서 분자들이 어떻게 쌓이는지(결정 패킹)에 의존합니다. 그런데 현재 정보로는 이것을 전혀 알 수 없습니다.

> 사람의 키를 예측하는데, 몸무게만 보고 나이·성별·유전자 정보는 전혀 안 보는 것과 같습니다.

**해결 방법:**

| 추가할 정보 | 쉬운 설명 | 기대 효과 |
|------------|-----------|-----------|
| MACCS Keys (167개) | 특정 작용기(OH, COOH 등)가 있는지 직접 확인 | 수소결합 패턴 파악 |
| RDKit Fingerprint | Morgan과 다른 방식의 구조 인코딩 | 상호 보완 |
| RDKit 기술자 200종 | 분자 복잡도, 대칭성 등 추가 정보 | 결정 패킹 간접 반영 |
| SHAP 기반 피처 선택 | 중요한 정보 300개만 선별 | 노이즈 제거 |

> SHAP = 각 피처가 예측에 얼마나 기여했는지 측정하는 방법

**예상 R² 향상: +0.05 ~ +0.10**

---

### 문제 3. 모델 하나로는 한계가 있다 (Agent 1 + Agent 4)

**앙상블(여러 모델 합치기):**

XGBoost + LightGBM + Random Forest 세 모델의 예측값을 평균 냅니다. 각 모델이 서로 다른 실수를 하기 때문에 평균을 내면 오차가 줄어듭니다.

**예상 R² 향상: CV 0.53 → 0.60~0.65**

**MLP(신경망) 추가:**

3~4층짜리 간단한 신경망을 추가하면 XGBoost가 잡지 못하는 비선형 패턴을 추가로 학습할 수 있습니다.

> MLP = Multi-Layer Perceptron, 여러 층으로 쌓인 신경망

**예상 R² 향상: +0.05 ~ +0.10**

---

## v2 개선 순서 (우선순위)

```
1단계 (데이터 정리)
  → 음수 MP 제거
  → log(MP) 타깃 변환
  → StratifiedKFold 적용

2단계 (피처 확장)
  → MACCS Keys + RDKit FP 추가
  → RDKit 기술자 200종 확장
  → SHAP으로 상위 300개 선별

3단계 (모델 앙상블)
  → XGBoost + LightGBM + RandomForest 가중 평균
  → 각 모델 Optuna 튜닝

4단계 (선택)
  → MLP 추가
```

---

## 예상 최종 성능

| 단계 | 예상 CV R² | 예상 Test R² |
|------|-----------|-------------|
| 현재 (base) | 0.5346 | 0.4831 |
| 1단계 후 | 0.58~0.62 | 0.55~0.60 |
| 2단계 후 | 0.63~0.68 | 0.60~0.65 |
| 3단계 후 | 0.68~0.72 | 0.65~0.70 |

> 이론적 상한: R² ≈ 0.80 (녹는점은 결정 구조·다형성 등 SMILES만으로 포착 불가한 정보 존재)

## 전문가 4인 계획서 평가 결과

### Reviewer 1 — 데이터 전략

**잘된 점:** 음수 MP 제거, 샘플 가중치 방향 타당.

**보완 필요:**
- 중복 샘플 제거, SMILES 유효성 검증 누락
- StratifiedKFold bin 설정: `pd.qcut(y, q=10)` 으로 균등 분포 보장 권장
- log 변환 후 예측 시 **exp() 역변환** 필수 — 역변환 없으면 결과값 단위 왜곡

**성능 전망 수정:**
> +0.20은 낙관적. Mordred 등 3D 기술자 없이는 달성 어려움. **현실 목표: R² 0.65~0.70**

---

### Reviewer 2 — 피처 엔지니어링

**잘된 점:** SHAP 기반 선별 방향 적절.

**보완 필요:**
- Morgan FP ↔ RDKit FP 간 중복률이 높아 실질 정보 증가가 제한적 → 사전 분산 필터링 후 합치기
- **SHAP 순서 주의**: 전체 피처로 1차 학습 → SHAP 계산 → 상위 300개 선별 → 재학습 순서를 반드시 지켜야 함 (순서 어기면 데이터 누수 발생)
- FP 3종 합산 약 4,300차원 + 기술자 200종은 과적합 위험 → PCA 또는 분산 필터링으로 사전 축소 권장

---

### Reviewer 3 — 모델 아키텍처

**잘된 점:** 소규모 데이터에 앙상블 방향 합리적.

**보완 필요:**
- XGBoost + LightGBM + RF는 **모두 트리 기반** → 실수 패턴이 비슷해 앙상블 다양성 제한
- SVR(서포트 벡터 회귀) 또는 Ridge 같은 **이질적 모델** 1개 추가 시 앙상블 효과 극대화
- MLP 추가 시 **StandardScaler 전처리 필수** (스케일링 없으면 학습 불안정)
- 가중치 결정: OOF(Out-of-Fold) 예측 기반 Ridge 스태킹 또는 균등 가중치가 소규모 데이터에 안전

> OOF = 각 Fold의 검증셋 예측값을 모아 메타 모델 학습에 사용하는 방법

---

### Reviewer 4 — 전체 실현 가능성 (시니어)

**순서 수정 제안:**
> SHAP top-300 선별은 **앙상블 완성 후**에 하는 게 더 정확한 피처 중요도 산출 가능

**단계별 효과/리스크:**

| 단계 | 기대 효과 | 리스크 |
|------|-----------|--------|
| 1단계 (데이터) | 중간 | **높음** — 음수 MP가 실은 유효한 데이터일 수도 있음, 무분별한 제거 시 저온 예측력 손실 |
| 2단계 (피처) | **가장 큼** | 중간 — 차원 폭발 주의 |
| 3단계 (앙상블) | 중간 | 낮음 |
| 4단계 (MLP) | 중간 | 중간 — 소규모 과적합 |

---

## 평가 반영 후 수정된 개선 순서

```
1단계 (데이터 정리) ← 리스크 주의
  → 중복 샘플 제거, SMILES 유효성 검증
  → 음수 MP: 원본 데이터 확인 후 수정 or 제거 (무조건 제거 금지)
  → log(MP) 타깃 변환 (예측 후 exp() 역변환 필수)
  → pd.qcut(y, q=10) 기반 StratifiedKFold 적용

2단계 (피처 확장) ← 효과 가장 큼
  → 분산 필터링으로 저분산 피처 사전 제거
  → MACCS Keys + RDKit FP 추가 (Morgan과 중복 고려)
  → RDKit 기술자 200종 확장
  → 고MP 샘플에 sample_weight 부여

3단계 (앙상블)
  → XGBoost + LightGBM + RF + SVR 가중 평균
  → 각 모델 Optuna 튜닝
  → SHAP top-300 선별 (앙상블 완성 후)

4단계 (선택)
  → MLP 추가 시 StandardScaler 전처리 필수
```
