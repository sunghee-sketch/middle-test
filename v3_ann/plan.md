# v3_ann 설계 계획서

## 왜 ANN(MLP)을 선택했는가?

v2까지는 XGBoost·LightGBM·RandomForest 같은 **트리 기반 모델**만 사용했습니다. 트리 모델은 피처 간의 조합 패턴을 잘 잡지만, **복잡한 비선형 관계**를 학습하는 데는 한계가 있습니다.

ANN(Artificial Neural Network, 인공신경망)은 여러 층(Layer)을 쌓아 데이터를 단계적으로 추상화하며, 트리 모델이 놓치는 고차원 비선형 패턴을 학습할 수 있습니다.

| 비교 항목 | 트리 모델 (XGBoost 등) | ANN (MLP) |
|----------|----------------------|-----------|
| 학습 방식 | 규칙 기반 분기 | 가중치 기반 함수 근사 |
| 비선형 패턴 | 제한적 | 강함 |
| 피처 스케일 민감도 | 낮음 | 높음 (StandardScaler 필수) |
| 소규모 데이터 | 유리 | 과적합 주의 |

---

## 모델 아키텍처

### MLP 구조

```
입력층 (2048 + 167 + ~200 = ~2400 차원)
    ↓
Linear(512) → BatchNorm → ReLU → Dropout(0.3)
    ↓
Linear(256) → BatchNorm → ReLU → Dropout(0.3)
    ↓
Linear(128) → BatchNorm → ReLU → Dropout(0.2)
    ↓
Linear(1) → 출력 (log1p(MP))
```

### 각 구성요소 역할

| 구성요소 | 역할 |
|---------|------|
| **BatchNorm** | 각 층의 출력을 정규화하여 학습 안정화 |
| **ReLU** | 비선형 활성화 함수 (음수 → 0) |
| **Dropout** | 학습 중 일부 뉴런을 무작위로 비활성화하여 과적합 방지 |
| **Weight Decay (1e-4)** | Adam 옵티마이저에 L2 정규화 추가 |

---

## 피처 엔지니어링

v2와 동일한 3종 피처를 사용합니다.

| 피처 | 차원 | 설명 |
|------|------|------|
| Morgan Fingerprint | 2048 | 분자 부분구조 (반경 2) |
| MACCS Keys | 167 | 작용기 존재 여부 (OH, COOH 등) |
| RDKit 기술자 | ~200 | 분자량, 극성, 복잡도 등 |
| **합계** | **~2400** | |

> **트리 모델과 달리 ANN은 피처 스케일에 민감합니다.**  
> StandardScaler로 전처리하여 평균 0, 표준편차 1로 정규화합니다.

## 학습 전략

### 타깃 변환
- `log1p(MP)` 로 변환하여 학습 → 예측 후 `expm1()` 역변환
- 고온 영역 분포 왜곡을 완화

### 샘플 가중치
- 상위 10% 고MP 샘플에 가중치 3.0 부여
- 데이터가 적은 고온 영역 학습 강화

### 조기 종료 (Early Stopping)
- `patience=20`: 검증 손실이 20 에포크 연속 개선 없으면 학습 중단
- 과적합 방지 및 불필요한 학습 시간 절약

### ReduceLROnPlateau
- 검증 손실이 10 에포크 동안 개선 없으면 학습률을 절반으로 감소
- 학습 후반부 세밀한 수렴 유도

### StratifiedKFold
- `pd.qcut(y, q=10)` 으로 MP 분포를 10구간으로 나눔
- 각 Fold에 고온·저온 샘플이 균등하게 배분되도록 보장

## 하이퍼파라미터

| 파라미터 | 값 | 선택 이유 |
|---------|-----|---------|
| Epochs | 150 (CV) / 300 (Final) | Early stopping으로 실제 종료 시점 자동 결정 |
| Batch size | 64 | 메모리와 학습 안정성의 균형 |
| Learning rate | 1e-3 | Adam 기본값, ReduceLROnPlateau로 자동 감소 |
| Weight decay | 1e-4 | L2 정규화로 과적합 방지 |
| Dropout | 0.3 / 0.3 / 0.2 | 앞 층일수록 강한 정규화 |

> v2와 달리 Optuna 튜닝을 사용하지 않습니다.  
> ANN은 에포크마다 학습이 진행되어 튜닝 시간이 매우 길어지므로,  
> 검증된 기본 하이퍼파라미터 + Early Stopping으로 대체합니다.

## 예상 성능

| 단계 | 예상 CV R² |
|------|-----------|
| base (XGBoost) | 0.5346 |
| v2 (앙상블) | 0.65~0.70 |
| v3_ann (MLP) | 0.60~0.68 |

> ANN은 데이터가 충분하지 않으면 트리 모델보다 낮을 수 있습니다.  
> 녹는점 데이터셋 크기(수천 개)에서는 앙상블이 일반적으로 유리합니다.
